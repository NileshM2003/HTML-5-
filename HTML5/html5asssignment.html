1. What are the new tags added in HTML5?

➢ HTML5 introduced several new elements and attributes that provide better semantics, structure, and functionality to web documents. Here are some of the key additions: Semantic Elements:

header: Defines a header for a document or section.
nav: Defines a set of navigation links.
section: Defines a section in a document.
article: Defines an independent, self-contained content.
aside>: Defines content aside from the content (like a sidebar).
footer>: Defines a footer for a document or section.
Multimedia Elements:

audio>: Embeds sound content in documents.
video>: Embeds video content in documents.
source>: Specifies multiple media resources for video>, audio>, and picture> elements.
Form Elements:

datalist>: Specifies a list of pre-defined options for input> elements.
output>: Represents the result of a calculation (like from a form).
progress>: Represents the progress of a task.
New Input Types:

input type="email">: Specifies an input field for email addresses.
input type="url">: Specifies an input field for URLs.
input type="number">: Specifies an input field for numbers.
input type="range">: Represents a control for selecting a value from a range.
input type="date">, input type="time">, input type="datetime-local">: For date and time input.
input type="color">: Allows users to select a color.
New Attributes:

placeholder: Provides a hint to the expected value of an input field.
autocomplete: Indicates whether the value of an input field can be automatically completed.
required: Specifies that an input field must be filled out before submitting a form.
autofocus: Specifies that an input field should automatically get focus when the page loads.
pattern: Specifies a regular expression that the input's value must match.
Canvas and SVG:

canvas>: Provides a drawing space for graphics on a webpage.
svg>: Defines vector-based graphics for the web.
These are some of the major additions brought by HTML5. They improve the semantics, accessibility, and functionality of web documents.

  • 3.Semantic element in HTML5?
    •  ➢ Semantic elements in HTML5 are elements that carry meaning about the content they contain. They provide a clearer structure to the HTML document, making it more understandable for both humans and machines (such as search engines and screen readers). Here are some commonly used semantic elements in HTML5:

    header>: Represents introductory content, typically containing headings, navigation links, or other elements related to the header of a document or section.
    nav>: Represents a section of navigation links, allowing users to navigate the website or move between different parts of the document.
    main>: Represents the main content of the document. It should not contain content that is repeated across multiple pages, such as site navigation or copyright information.
    section>: Represents a thematic grouping of content, typically with a heading. It helps to organize content into distinct sections.
    article>: Represents a self-contained piece of content, such as a blog post, article, or comment. It can be distributed independently from the rest of the document.
    aside>: Represents content that is tangentially related to the content around it, such as sidebars, pull quotes, or advertisements.
    footer>: Represents the footer of a document or section, typically containing metadata, copyright information, or contact details.

    4.Canvas and SVG tags


    canvas>:
    The canvas> element provides a drawable region defined in HTML code with JavaScript.
    It is used for dynamic, scriptable rendering of 2D shapes, graphics, and animations.
    The drawing is done programmatically via JavaScript using a set of drawing APIs.
    The content of a canvas is not accessible to search engines or screen readers unless explicitly provided through alternate means.
    svg>:
    The svg> element defines a graphics container that can contain shapes, paths, text, and other graphical elements.
    SVG stands for Scalable Vector Graphics, and it's based on XML syntax.
    SVG content is part of the DOM (Document Object Model), so it can be easily styled and manipulated using CSS and JavaScript.
    SVG graphics are scalable and resolution-independent, making them ideal for creating vector graphics, icons, charts, and interactive graphics.
    SVG content is accessible to search engines and screen readers.
